氏名漢字推計API
##############################
 漢字推計機能は，読みであるカタカナ姓名入力し，その読みから推測される漢字・アルファベット姓名を出力するものである。

 本機能は，姓名辞書などを用いず，AIのみを用いた推計である。AIで推計した確率の高い順に出力する。

 カナ姓名と最大漢字姓名返却個数を入力し，漢字姓名の一覧の結果をJSONで取得。最大返却個数は省略可能であり，システムデフォルト値の最大個数５で返す。

APIのURL
===================

 https://api.kktg.digital.go.jp/v1/kanji

 * v1
  
  漢字カナ突合モデルのバージョンによってここが変わる

入力パラメタ
====================

 * kana
  
   カナ姓名を入力。:doc:`data` を参照のこと。

 * max
  
   最大カナ姓名返却個数。1から10の整数を指定する。


入力例
-----------------

 * 姓名

  例  https://api.kktg.digital.go.jp/v1/kanji?kana=ヤマダ　タロウ&max=10

 * 姓のみ

  例 https://api.kktg.digital.go.jp/v1/kanji?kana=ヤマダ

 * 名のみ
  
  例 https://api.kktg.digital.go.jp/v1/kanji?kana=タロウ


出力例
--------------

 出力はHTTPボディにJSON形式で出力される。

 .. code-block:: json

    {
        "response": "OK",  // コード参照
            "result":{
                "status":"OK",
                "kanji",[
                    "山田　太郎",
                    "山田　太朗",
                       // 最大max個出力
                ]
                "probability":[
                    0.8000104567944917,   // 山田　太郎 の確率
                    0.0009880204100894683, // 山田　太朗 の確率
                  // 最大max個出力
                ]  
            }
        }
    }

 * repsonse:

    :doc:`code` 参照
    
 * status

  + OK
  
   カナ姓名から漢字姓名が得られた。

  + NG
  
   カナ姓名から漢字姓名が得られなかった。　kanjiは0の配列となる
   
 * kanji
 
  モデルから取得した漢字姓名一覧。最大max個，ない時（statusがNGの時）は０の配列
